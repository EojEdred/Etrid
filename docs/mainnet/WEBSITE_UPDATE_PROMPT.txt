## What Was Added

### Key Concepts Explained:

**1. Three-Layer Architecture**
- Layer 1: FlareChain (1,000 TPS, Directors 1-5, GRANDPA finality)
- Layer 2: PBCs (70,000 TPS, Validators 6-21, checkpoint every 256 blocks)
- Layer 3: Lightning Bloc (100,000+ TPS, off-chain, batch every 5 min)
- Total: 171,000+ TPS network capacity

**2. PBC → FlareChain State Synchronization**
- Checkpoints submitted every 256 blocks (~51 minutes)
- Merkle roots only (32 bytes per PBC)
- 7-day fraud proof challenge period
- GRANDPA finality in 2 blocks (24 seconds)

**3. Lightning Bloc → PBC Settlement**
- Batch 1,000 transactions or 5-minute timeout
- Compression: 150 KB → 105 KB (30%)
- Settlement to PBC via Merkle root
- Total latency: ~56 minutes to FlareChain finality

**4. Validator Role Separation**
- Flare Nodes: Directors 1-5 (FlareChain validators)
- Validity Nodes: Validators 6-21 (PBC validators)
- Clear architectural principle: Only Directors can be Flare Nodes
