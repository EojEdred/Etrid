Why SSL/WSS?

Modern browsers block insecure connections from HTTPS websites to HTTP/WebSocket (WS) endpoints. This is called "mixed content blocking" and is a security feature.

‚ö†Ô∏è Problem: https://etrid.org (HTTPS) cannot connect to ws://your-node:9944 (WS)
‚úÖ Solution: Use WSS (WebSocket Secure) and HTTPS endpoints with SSL certificates

Setup Options

Option 1: Use Let's Encrypt (Recommended - Free)

Let's Encrypt provides free SSL certificates that auto-renew.

# Install Certbot
sudo apt update
sudo apt install certbot

# Get SSL certificate for your domain
sudo certbot certonly --standalone -d node.etrid.org

# Certificates will be saved to:
# /etc/letsencrypt/live/node.etrid.org/fullchain.pem
# /etc/letsencrypt/live/node.etrid.org/privkey.pem

Option 2: Configure Nginx as Reverse Proxy

Use Nginx to handle SSL and proxy requests to your node.

# Install Nginx
sudo apt install nginx

# Create Nginx configuration
sudo nano /etc/nginx/sites-available/etrid-node

# Add this configuration:
server {
    listen 443 ssl http2;
    server_name node.etrid.org;

    ssl_certificate /etc/letsencrypt/live/node.etrid.org/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/node.etrid.org/privkey.pem;

    # WebSocket RPC (WSS)
    location /ws {
        proxy_pass http://localhost:9944;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # HTTP RPC (HTTPS)
    location /rpc {
        proxy_pass http://localhost:9933;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

# Enable the site
sudo ln -s /etc/nginx/sites-available/etrid-node /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

Your New Secure Endpoints

After SSL setup, update your apps to use:

  • WSS RPC: wss://node.etrid.org/ws
  • HTTPS RPC: https://node.etrid.org/rpc

Firewall Rules

Open ports for HTTPS/WSS:

# Allow HTTPS (443)
sudo ufw allow 443/tcp

# Allow HTTP (for Let's Encrypt renewal)
sudo ufw allow 80/tcp

# Internal ports (keep blocked from public)
# 9933 and 9944 should only be accessible via Nginx proxy

Auto-Renewal Setup

Let's Encrypt certificates expire after 90 days. Set up auto-renewal:

# Test renewal
sudo certbot renew --dry-run

# Add to crontab for auto-renewal
sudo crontab -e

# Add this line:
0 3 * * * certbot renew --post-hook "systemctl reload nginx"

Testing Your Setup

Verify SSL is working:

# Test WSS connection
wscat -c wss://node.etrid.org/ws

# Test HTTPS RPC
curl -H "Content-Type: application/json" \
  -d '{"id":1, "jsonrpc":"2.0", "method": "system_health"}' \
  https://node.etrid.org/rpc

# Check SSL certificate
openssl s_client -connect node.etrid.org:443

Troubleshooting

Certificate Errors

  • Ensure port 80 is open for Let's Encrypt validation
  • Check DNS points to your server IP
  • Wait 5-10 minutes for DNS propagation

Connection Refused

  • Verify Nginx is running: sudo systemctl status nginx
  • Check Nginx logs: sudo tail -f /var/log/nginx/error.log
  • Ensure FlareChain node is running on ports 9933/9944

Mixed Content Still Blocked

  • Clear browser cache (Ctrl+Shift+Del)
  • Check browser console for errors (F12)
  • Verify using wss:// not ws://
